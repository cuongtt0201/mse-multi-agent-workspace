AI Workspace là gì?

Một trợ lý đa tác vụ tinh gọn cho không gian làm việc số. Sở hữu tính năng Platform-agnostic — hỗ trợ tích hợp linh hoạt với nhiều nền tảng (Google Workspace, Slack, Notion...); hiện đã hoàn thiện kết nối Google Workspace. Hệ thống hoạt động trên kiến trúc đa Agent phi tập trung, giao tiếp tự nhiên bằng tiếng Việt, tích hợp trí nhớ ngữ cảnh dài hạn và cơ chế duyệt thủ công (HITL) cho các hành động bảo mật nhạy cảm.

Vấn đề & Giải pháp

Thực trạng

Công việc số bị phân mảnh, thiếu tự động hóa.

  • Tìm thông tin App này, copy sang App khác thủ công.
  • Thao tác lặp lại — lãng phí thời gian, dễ sai sót.
  • Trợ lý AI hiện tại chỉ "trả lời chat", không hành động thực tế.

Giải pháp

Điều hướng mọi tác vụ bằng ngôn ngữ tự nhiên.

  • 1 lệnh → chuỗi hành động: tự lập kế hoạch đa dịch vụ.
  • Liên kết dữ liệu ngầm: agent này truyền kết quả cho agent kia, không hỏi lại user.
  • Trí nhớ ngữ cảnh: "Gửi báo cáo cho Tính" → tự tra email của Tính, ánh xạ vào tham số Gmail Agent.

Kiến trúc & Cách hoạt động

Choreography phi tập trung — hai mặt phẳng tín hiệu & ký ức, worker máy trạng thái, và bộ nhớ đồ thị.

Kiến trúc: Event-Driven Choreography

Không có bộ não trung tâm. Các Agent là thực thể độc lập, chia sẻ trạng thái & kết quả thông qua Blackboard Redis.

  • Control Plane (RabbitMQ): bus sự kiện chở tín hiệu Task/Result — chỉ định danh, không chứa data thô.
  • Data Plane (Redis Blackboard): ký ức chung, key thống nhất (HASH per-field) — mọi agent đọc/ghi song song an toàn.
  • Platform Agnostic: mở rộng nền tảng mới qua Token Registry chung, không sửa lõi.

Điểm mạnh kiến trúc — Scale & Bất đồng bộ

Khả năng Scale Ngang

Thiết kế phân tán hoàn hảo cho phép scale-out độc lập:

  • ✦ Microservices độc lập: Gateway, Intent Planner và các Agent con tách biệt hoàn toàn, không phụ thuộc mã nguồn của nhau.
  • ✦ Đa Pod (K8s): Sẵn sàng nhân bản (scale-out) riêng lẻ từng agent worker có tải cao mà không ảnh hưởng hệ thống chung.

Bất đồng bộ phi tập trung

Hạn chế điểm nghẽn (bottleneck) của Orchestrator trung tâm:

  • ✦ Cơ chế Choreography: Tự kích hoạt tác vụ dựa trên event bus, không phụ thuộc vòng lặp LLM trung tâm quản lý luồng.
  • ✦ Song song & Không chặn: Các Agent con (Gmail, Drive) thực thi đồng thời, tự đỗ chờ (parked) khi thiếu dependencies.

Cô lập lỗi & Chịu lỗi

Hạn chế ảnh hưởng lỗi dây chuyền lên toàn hệ thống:

  • ✦ Vận hành độc lập: Mỗi agent worker chạy riêng biệt. Nếu một agent worker gặp sự cố hoặc ngừng hoạt động (chết), toàn bộ hệ thống vẫn hoạt động bình thường.
  • ✦ Cô lập sự cố: Sự cố chỉ nằm cục bộ tại một agent duy nhất, không làm sập (crash) lan truyền sang Gateway hay các agent khác.

Gateway API — Cổng điều phối & Bảo mật

ai-workspace-gateway/backend

Hệ thống Gateway phức hợp (FastAPI)

  • OAuth2 Broker (routes_oauth.py · routes_connect.py): Quản lý luồng bắt tay, callback OAuth2 và pairing tokens đa nền tảng.
  • Chat & SSE Stream (routes_chat.py · routes_sse.py): Xử lý nghiệp vụ chat (hơn 500 dòng logic) và multiplexing stream SSE (heartbeats, logs, tasks, interactive cards).
  • Vault Credential (credential_manager.py): Mã hóa đối xứng Fernet lưu DB, tự động refresh token định kỳ trước hạn 5 phút tránh race conditions.
  • Bảo mật & Sync (rate_limit.py · memory_worker.py): Tích hợp JWT auth, rate limiting và lắng nghe Event Bus đồng bộ trạng thái hội thoại.

Intent Gate & Trí nhớ Ngữ cảnh

ai-workspace-intent/backend/intent

Lập kế hoạch & Sửa lỗi cạnh

Phân rã ngôn ngữ tự nhiên thành đồ thị tác vụ (Task DAG) cho các agent:

  • FSM Routing (gate.py · main.py): Tiếp nhận từ Event Bus, phân loại chat/công việc, khôi phục lịch sử phiên chạy từ Redis Blackboard.
  • Task DAG Gen (planner.py · builder.py): Gọi Gemini sinh sơ đồ tác vụ phụ thuộc chéo thông qua biến truyền nhận needs_from.
  • Plan Repair (planner.py): Kiểm tra liên kết dữ liệu thô, tự động sửa cạnh đồ thị thiếu thông tin trước khi đẩy vào hàng đợi thi hành.
ai-workspace-intent/backend/memory

Trí nhớ 3 lớp định ngữ cảnh

Bơm thông tin cá nhân hóa giúp LLM điền chính xác tham số tác vụ:

  • Memory Fusion (retriever.py): Tổng hợp Manual Memory (khai báo tĩnh), Session Summaries (tóm tắt chat ngắn hạn) và mối liên hệ thực thể Graph RAG.
  • Giải nghĩa thực thể: Tra cứu quan hệ người-file-kênh trong cơ sở dữ liệu đồ thị Graph RAG để suy luận danh từ riêng trong câu chat.
  • Tự động điền tham số: Điền sẵn email, ID thư mục vào đối số gọi tool mà worker agent không cần hỏi lại người dùng.

Hệ thống các Agent con chuyên biệt

Mỗi agent phụ trách một dịch vụ riêng biệt, hoạt động độc lập và bất đồng bộ:

  • Drive Agent: Định vị tài liệu, đọc/ghi file, tự động nới rộng từ khóa tìm kiếm (không dấu/có dấu).
  • Gmail Agent: Quản lý danh sách email, soạn thảo thư nháp (drafts) và gửi thư theo yêu cầu.
  • Calendar Agent: Tra cứu lịch biểu cá nhân, lên lịch hẹn mới và tự động chèn liên kết Google Meet.
  • Docs Agent: Đọc tệp tin, trích xuất văn bản và tóm tắt nội dung tài liệu.
agent/graph.py · LangGraph FSM

FSM tất định (Classify ➔ Execute ➔ Reflect)

Tất cả các Agent con đều chạy chung một mô hình đồ thị trạng thái tất định:

  • Classify & Execute: Gọi LLM 1-shot bóc tham số cấu trúc, chạy Playbook tương tác trực tiếp API.
  • Tự sửa (Reflect): Tự nới query bằng toán tử OR (ví dụ: "báo cáo" OR "bao cao") và chạy lại nếu search rỗng (tối đa 2 lần).
  • Đỗ chờ (Parked state): Tự hoãn chạy nếu Blackboard báo thiếu dữ liệu từ Agent khác, ghi kết quả lên Redis trước khi báo qua RabbitMQ.
classify ──► execute ──► reflect ──► (retry ≤2) ──► ask_user

Frontend SSE — Giao diện thời gian thực

React · routes_sse.py

  • Reload trang khôi phục lại run + thẻ duyệt từ Blackboard.

Mở rộng Kết nối API & LLM

Nâng cao khả năng kết nối và tối ưu chi phí vận hành:

  • Tích hợp thêm nền tảng API mới: Mở rộng registry kết nối sang **GitLab, GitHub, Microsoft 365, Slack, Notion...** giúp điều khiển trọn vẹn không gian Dev/Ops và văn phòng.
  • Chuỗi liên kết tác vụ đa nền tảng: Thực thi chuỗi nghiệp vụ đan chéo phức tạp từ các nền tảng khác nhau trong cùng một câu lệnh ngôn ngữ tự nhiên duy nhất.
  • Tích hợp SLM nội bộ (Self-hosted/Offline): Hỗ trợ chạy các mô hình ngôn ngữ nhỏ tinh gọn trên máy chủ riêng giúp tối ưu hóa chi phí token và bảo mật dữ liệu doanh nghiệp tuyệt đối.
